Programming Fundamentals

1. Software & Programs

Program: Set of instructions written in a programming language.

Software: Collection of integrated programs performing specific tasks.

Language Translators: Convert code to machine language (assembler, compiler, interpreter).

Qualities of Good Programs

  • Readable and understandable.
  • Error-free and reliable.
  • Portable and flexible.
  • User-friendly interface.

2. Programming Languages

Low-Level Languages

Machine Language (1GL): Binary code (0s/1s).

  • Advantages: Fast execution, no translator.
  • Disadvantages: Machine-dependent, hard to debug.

Assembly Language (2GL): Uses mnemonics (e.g., ADD, SUB).

  • Requires an assembler.
  • Advantages: Faster than high-level languages.
  • Disadvantages: Machine-dependent, complex syntax.

High-Level Languages

Procedural (3GL): Focuses on procedures (e.g., C, FORTRAN).

Object-Oriented (4GL): Focuses on data (e.g., C++, Java).

Natural Language (5GL): Human-like syntax (e.g., PROLOG).

3. Language Translators

Translator Function Examples
Assembler Converts assembly to machine code. NASM
Compiler Translates entire code at once. C, C++, Java
Interpreter Translates line-by-line. Python, PHP, BASIC

4. Program Design Tools

Algorithm

Step-by-step problem-solving procedure.

Step 1: Start  
Step 2: Read P, T, R  
Step 3: Calculate I = (P*T*R)/100  
Step 4: Display I  
Step 5: Stop

Flowchart

Visual representation using symbols:

  • Oval: Start/End
  • Rectangle: Process
  • Diamond: Decision
  • Parallelogram: Input/Output

5. C Programming Language

Features

  • Middle-level language (supports low/high-level features).
  • Structured, portable, efficient for system programming.
  • Rich library functions (e.g., printf(), scanf()).

Basic Structure

#include <stdio.h> 
void main() {      
  // Code
}

Data Types

  • int: Integer (2 bytes).
  • float: Decimal (4 bytes).
  • char: Single character (1 byte).
  • void: No value.

6. Common Programs

Sum of Two Numbers

#include <stdio.h>
void main() {
  int a, b, s;
  scanf("%d %d", &a, &b);
  s = a + b;
  printf("Sum: %d", s);
}

Check Palindrome Number

int n, rev = 0, original;
scanf("%d", &n);
original = n;
while (n != 0) {
  rev = rev * 10 + n % 10;
  n /= 10;
}
if (original == rev) printf("Palindrome");

7. Key Terms

  • Syntax: Rules for writing code.
  • Semantics: Meaning of code.
  • Errors:
    • Syntax: Incorrect code structure.
    • Logical: Flawed logic.
    • Runtime: Errors during execution (e.g., division by zero).
